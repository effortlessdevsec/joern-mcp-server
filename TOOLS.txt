# Joern MCP Server ‚Äì All Tools

After code changes: run `npm run build`, then reload MCP in Cursor (or restart Cursor) so the client sees these tools.

Env (see BACKEND.md): `JOERN_SERVER_URL` ‚Üí script tools POST `/query-sync` in-memory (no temp file). `JOERN_MCP_HTTP_ONLY=1` ‚Üí scripts require URL; `joern_scan` / `joern_parse` / `joern_export` / `joern_slice` / list tools error (no HTTP API).

0a. joern_http_check
    - query? (default `cpg.graph.nodeCount`)
    - POST /query-sync health probe; requires `JOERN_SERVER_URL` (ignores `JOERN_MCP_FORCE_SPAWN`).

0b. joern_http_query
    - query (required)
    - Raw CPGQL via POST /query-sync (curl-equivalent). `plotDot*` often yields empty stdout + SVG via Graphviz ‚Äî see tool description.

    Plots over HTTP: see **`PLOT-HTTP.md`**. Prefer `cpg.method.take(1).dotDdg.foreach(println)` (DOT in stdout) instead of `plotDotDdg` when you need text in MCP.

1. joern_run_script
   - script, cpgPath?, params?
   - Run any Joern Scala script (full CPG API).

2. joern_find_calls
   - cpgPath, methodFullNameRegex?, callNameRegex?, whereArgumentIndex?, whereArgumentCodeRegex?
   - Find call nodes (e.g. Log calls where arg 1 matches .*pass.*).

3. joern_list_call_names
   - cpgPath, methodFullNameRegex?
   - List distinct call names.

4. joern_methods
   - cpgPath, nameRegex?, fullNameRegex?
   - List methods (name, fullName, filename).

5. joern_import_code
   - inputPath, language?, runOssDataflow? (default true)
   - Joern importCode.<language>(path); optional run.ossdataflow; returns projectPath.

5b. joern_importcode
   - Same args and behavior as joern_import_code (duplicate name for importCode discoverability).

6. joern_query_flows
   - cpgPath, sourceCallName, sinkCallNamePattern
   - sourceMatch?, sinkMatch?, sourceMethodFullNameRegex?, sinkMethodFullNameRegex?
   - sourceArgIndex?, sinkArgIndex?, methodRegex?, fileRegex?
   - Data flows source ‚Üí sink.

7. joern_list_methods
   - cpgPath
   - List all method names.

8. joern_dump_graphs
   - cpgPath, sourceCallName, sinkCallNamePattern
   - outputDir?, graphTypes?, sourceMatch?, sinkMatch?, sourceArgIndex?, sinkArgIndex?
   - methodRegex?, fileRegex?, dumpFlowDot?, flowDotDir?
   - Dump AST/CFG/PDG .dot for methods in flows.

9. joern_full_flow_graph
   - cpgPath, sourceCallName, sinkCallNamePattern
   - outputDir?, emitSVG?, sourceMatch?, sinkMatch?, sourceArgIndex?, sinkArgIndex?
   - methodRegex?, fileRegex?
   - One merged DOT + SVG for all flows.

10. joern_scan
    - inputPath, language?, overwrite?, store?, names?, tags?, maxCallDepth?
    - `joern-scan` CLI on a source tree (query DB rules).

11. joern_scan_list_queries
    - (optional `_` ‚Äî leave empty)
    - `joern-scan --list-query-names`

12. joern_scan_list_languages
    - (optional `_` ‚Äî leave empty)
    - `joern-scan --list-languages`

13. joern_parse
    - inputPath?, outputCpgPath?, language?, listLanguages?, noOverlays?
    - `joern-parse`: write CPG to disk (default under MCP_OUT). Or listLanguages only.

14. joern_export
    - cpgPath, repr?, format?, outputDir?
    - `joern-export`: ast/cfg/ddg/cdg/pdg/cpg14/cpg/all ‚Üí dot/graphml/graphson/neo4jcsv (out dir must not exist; default fresh dir under MCP_OUT).

15. joern_slice
    - cpgPath, mode (data-flow | usages), outputPath?, plus filters (sinkFilter, sliceDepth, fileFilter, methodNameFilter, ‚Ä¶)
    - `joern-slice`: JSON slices.

## Joern execution backend (modular)

**Script tools** (1‚Äì9, import 5‚Äì5b, query/list/dump/flow): **prefer `JOERN_SERVER_URL`** ‚Üí **`POST /query-sync`** on `joern --server`. If **`JOERN_SERVER_URL` is not set**, they **fall back** to **`joern [cpgPath] --script <temp.sc>`**. Optional **`JOERN_MCP_FORCE_SPAWN=1`** forces spawn even when the URL is set.

**Direct HTTP** (0a‚Äì0b): **`joern_http_check`** / **`joern_http_query`** always POST to **`JOERN_SERVER_URL/query-sync`** (even if `JOERN_MCP_FORCE_SPAWN` is set).

**CLI tools** (10‚Äì15): always **spawn** `joern-cli/joern-scan`, `joern-parse`, `joern-export`, `joern-slice` ‚Äî Joern has no HTTP API for these.

| Env var | Purpose |
|---------|--------|
| `JOERN_SERVER_URL` | **Set for production use:** script tools use **`POST .../query-sync`**. Unset ‚Üí spawn fallback. |
| `JOERN_SERVER_USER` / `JOERN_SERVER_PASSWORD` | HTTP Basic auth for the Joern server (aliases: `JOERN_SERVER_BASIC_USER` / `JOERN_SERVER_BASIC_PASS`). |
| `JOERN_SERVER_TIMEOUT_MS` | Max wait for `/query-sync` (default `300000`). |
| `JOERN_MCP_FORCE_SPAWN` | `1` / `true` / `yes` ‚Üí always spawn scripts (ignore URL; e.g. for `joern_run_script` **params**). |
| `JOERN_MCP_DEBUG` | `1` / `true` / `yes` ‚Üí full stderr trace: startup env, `[executor]` routing, `[http]` POST + responses, `[spawn]` CLI. See BACKEND.md. |
| `JOERN_MCP_DEBUG_QUERY_MAX` | Optional; chars of query preview in logs (default 2000). |
| `JOERN_HOME` | Joern install dir (**required** for `joern-cli/*` binaries unless they are on `PATH`). |

**HTTP notes (script tools):**
- Start Joern with the CPG you want, e.g. `joern --server --runBefore "run.ossdataflow" /path/to/cpg.bin`
- Tool `cpgPath` is **not** passed to the server per request; stdout includes a reminder if you still pass it.
- `joern_run_script` **`params`** are **not** forwarded over HTTP (use **`JOERN_MCP_FORCE_SPAWN=1`** or embed values in the script).
- **`joern_scan`**, **`joern_scan_list_*`**, **`joern_parse`**, **`joern_export`**, **`joern_slice`** always use **spawn**, not HTTP.

See `BACKEND.md` and `docs/JOERN-HTTP-SERVER-GUIDE.md` in the Joern repo.

**Source layout:** `src/create-server.ts` + `src/tools/*.ts` (one file per MCP tool), `src/joern/*` (spawn/HTTP executor), `src/flow-graph/merge-dot.ts`. Full table in `BACKEND.md`.
